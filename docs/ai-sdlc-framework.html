01
The Lifecycle

Eight-Phase AI-SDLC

Traditional SDLC assumes human developers hold context across sessions. AI-native development requires explicit context engineering at every phase transition.

📐
Initialize
Seed the project brain from documentation. Ingest SRS, architecture specs, diagrams. Establish governance guardrails.
🗺️
Plan
Architecture discovery graded by system complexity. Decompose into epics with dependency mapping and sprint candidates.
🎯
Sprint
Define scope, tasks, acceptance criteria, and deliverables. Task dependencies and file scopes enable parallel execution.
⚡
Execute
Sequential or wave-based parallel execution. Each task runs under governance constraints with inline security scanning.
✅
Verify
Blind verification against acceptance criteria. Optional DAST scanning for runtime vulnerabilities.
🔍
Review
Isolated specialist agents review independently. Code, security, architecture, QA — each blind to implementation reasoning.
🧠
Learn
Compress sprint into summaries. Extract patterns with trust-tiered approval. Record architecture decisions.
🔒
Scan
SAST, dependency audit, secrets detection at every phase. Configurable enforcement posture: strict, moderate, or permissive.
CONTINUOUS LOOP
Initialize → Plan → Sprint → Execute → Verify → Review → Learn → Sprint → ...
02
Architecture

Three-Layer Governance Model

Separating permanent governance from temporary execution prevents context rot and ensures consistency across AI agent sessions.

Governance Layer Always Active

The project's immune system. Holds architectural truth, coding standards, specialist agents, and quality hooks. Never modified by execution — only by explicit human decision.

Project Brain (Vision, Architecture, Domain, NFRs) 7 Specialist Agents Coding Standards (per language) Quality Hooks Architecture Decision Records Security Posture
Context Layer Traffic Control

Assembles minimal, relevant context per phase. Prevents token waste and context starvation through strict budgets, freshness policies, and priority-based dropping.

Context Packs (Planning, Execution, Review) Token Budgets (4,300–5,800 tokens) Inclusion / Exclusion Policies Freshness & Staleness Thresholds Compression Policies
Execution Layer Sprint-Scoped

Operates inside governance boundaries. Plans, implements, verifies, and learns. Can propose changes but never silently overwrite standards or decisions.

Sprint Workflows (Plan, Execute, Verify) Task Agents (Planner, Executor, Verifier) Wave-Based Parallel Execution Execution Ledger Security Scanner

Security

Multi-Layer Security Integration

Security scanning is not an afterthought — it's embedded at every lifecycle phase with configurable enforcement thresholds.

Strict
HIGH/CRITICAL findings block task completion. MEDIUM generates warnings. Zero tolerance for known vulnerabilities in production code.
Moderate
HIGH/CRITICAL generate warnings. All findings logged. Developer decides. Balanced approach for active development phases.
Permissive
All findings logged only. Nothing blocks. Maximum velocity during prototyping. Switch to strict before production.

06
Organizational Learning

Trust-Tiered Pattern Recognition

The system learns from three channels with different trust levels, approval gates, and expiry policies. Patterns are reinforced through application, not just collection.

01

Explicit Teaching

Human directly teaches the system a pattern or principle. No approval gate needed — the human said it. Patterns persist indefinitely.

HIGH TRUST
Evergreen · 0.9 confidence
02

Sprint Review Extraction

Patterns extracted from completed sprints. Inline approval — human confirms or rejects each pattern before it's saved. 90-day TTL with reinforcement.

MEDIUM TRUST
90-day TTL · 0.7 confidence
03

Log Analysis

Candidate patterns extracted from production logs and error data. Staged for batch review — never auto-applied. 30-day TTL if approved.

LOW TRUST
30-day TTL · 0.5 confidence
04

Pattern Lifecycle

Patterns that are applied successfully get reinforced (+confidence, TTL reset). Patterns that expire surface for audit. 50-pattern cap forces pruning — quality over quantity.

🔄
Reinforce → Expire → Audit
